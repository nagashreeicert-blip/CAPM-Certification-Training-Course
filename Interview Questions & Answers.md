A structured set of **CAPM interview questions and answers** for beginners, intermediate, and advanced professionals preparing for project management roles.

## üü¢ Beginner-Level Questions & Answers (10)

1. **What is a Project?**  
A project is a temporary endeavor undertaken to create a unique product, service, or result.

2. **What is Project Management?**  
The application of knowledge, skills, tools, and techniques to project activities to meet objectives.

3. **Who is a Stakeholder?**  
Anyone who is affected by or can affect the outcome of a project.

4. **What is Project Scope?**  
All the work required to complete a project successfully.

5. **What is a Deliverable?**  
A tangible or intangible output produced by a project.

6. **What is a Work Breakdown Structure (WBS)?**  
A hierarchical breakdown of project deliverables into smaller, manageable components.

7. **What is Risk in project management?**  
An uncertain event that can affect project objectives positively or negatively.

8. **What is the difference between Project and Operations?**  
Projects are temporary; operations are ongoing and repetitive.

9. **Name the 5 Project Management Process Groups.**  
Initiating, Planning, Executing, Monitoring & Controlling, Closing.

10. **What is a Milestone in project management?**  
A significant point or event in the project timeline.

---

## üü° Intermediate-Level Questions & Answers (10)

1. **What is a Project Charter?**  
A document that formally authorizes a project and defines objectives, scope, stakeholders, and authority.

2. **What is a Risk Register?**  
A document listing all identified risks, their assessment, and planned responses.

3. **What is Earned Value (EV)?**  
A metric to measure work actually completed compared to planned work.

4. **Explain Critical Path Method (CPM).**  
The sequence of project activities that determines the shortest project duration.

5. **What is a Change Request?**  
A formal proposal to modify any aspect of a project.

6. **Explain Resource Leveling.**  
Adjusting the project schedule to resolve over-allocated resources without changing scope.

7. **What is a Baseline?**  
A reference point for scope, schedule, or cost used to measure project performance.

8. **What is a Kickoff Meeting?**  
The official meeting to start the project and align stakeholders.

9. **Difference between Issue and Risk?**  
- Issue: Current problem affecting the project.  
- Risk: Potential future event that may impact the project.

10. **What is a RAID Log?**  
A tool to track **Risks, Assumptions, Issues, and Dependencies** in a project.

---

## üî¥ Advanced-Level Questions & Answers (10)

1. **What is a RACI Chart?**  
A responsibility assignment matrix defining **Responsible, Accountable, Consulted, Informed** roles.

2. **What is Quantitative Risk Analysis?**  
Using numerical methods to analyze the impact and probability of risks.

3. **Explain Rolling Wave Planning.**  
Planning technique where near-term work is planned in detail, future work at a higher level.

4. **What is Hybrid Project Management?**  
Combining **Agile and Waterfall** approaches to leverage benefits of both.

5. **What is Cost of Quality (COQ)?**  
The total cost of ensuring quality, including prevention, appraisal, and failure costs.

6. **What is a Change Control Board (CCB)?**  
A group responsible for reviewing, approving, or rejecting change requests.

7. **Explain Monte Carlo Simulation in project management.**  
A statistical technique to estimate probable project outcomes by simulating scenarios multiple times.

8. **What is Procurement Management?**  
Acquiring goods and services from external suppliers and managing contracts.

9. **What is a Program Management Plan?**  
A plan describing how a program will be executed, monitored, and controlled, including governance, roadmap, and benefits realization.

10. **How do you handle conflicts in a project team?**  
By identifying the source, encouraging open communication, facilitating negotiation, and using conflict resolution techniques (e.g., collaboration, compromise, or escalation if needed).
